<Note>
  **LedgerShield ControlBench** evaluates whether an AI agent can run a defensible enterprise control regime — investigate under budget, trigger controls, absorb delayed evidence, withstand adversarial pressure, and submit an auditable proof-carrying decision. It is **not** a one-shot classification benchmark.
</Note>

## What it is

LedgerShield is an [OpenEnv](https://github.com/BiradarScripts/Meta-s-LedgerShield/blob/main/openenv.yaml)-compatible FastAPI environment that frames **invoice fraud investigation** as an *Adversarial Sequential Hypothesis Testing Game (ASHTG)* over partially-observable state, persistent institutional memory, and a deterministic falsifier.

## Why this matters

Real-world payment fraud is expensive and operationally messy. The FBI IC3 2023 report counts **21,489 BEC complaints and more than $2.9B in reported losses**, with total cybercrime losses exceeding **$12.5B**. LedgerShield turns that risk surface into an agent benchmark focused on **safe decision-making**, **evidence quality**, and **control discipline** instead of one-shot classification.

<Tip>
  Public mode is **blind by default** — SPRT, VoI rankings, and reward-machine scaffolding stay hidden from the acting agent. Run with `LEDGERSHIELD_TRACK_MODE=instrumented` for diagnostics.
</Tip>
